Why SVM generalizes better than a Perceptron.

This page turns the notebook’s core idea into a visual story: both models can separate data, but SVM keeps a safe margin from the boundary, which makes it more stable when the data changes or when outliers and noise appear.

Perceptron

Boundary follows training points No margin objective

The perceptron is satisfied when the data is separable. That makes it fast and elegant, but it can become sensitive to borderline points and outliers.

SVM

Maximizes margin Supports better stability

SVM searches for a separator with the widest possible cushion between classes. That cushion is what makes the model more resilient to small perturbations in the data.

What the page communicates

Perceptron is a correct-separation model; SVM is a correct-separation model with a built-in preference for wider margins. That distinction matters when the data is imperfect, noisy, or slightly shifted.
